Instalasi & Konfigurasi BIND9 (DNS Server)

Panduan lengkap â€” from preparing netplan to setting up forward & reverse zones, checks and troubleshooting.

Persiapan

  • OS: Ubuntu/Debian (tested on Ubuntu 22.04)
  • Akses root / sudo
  • VM: prepare bridged and host-only if needed

Update & install

System Update BASH 
sudo apt update
sudo apt upgrade -y
sudo apt install vim curl net-tools -y

Disable cloud-init (if using cloud images)

Cloud-init Configuration BASH 
sudo nano /etc/cloud/cloud.cfg.d/99-installer.cfg
# add:
network: {config: disabled}
# save and reboot if needed

Netplan static IP example

Netplan Configuration YAML 
sudo tee /etc/netplan/01-netcfg.yaml > /dev/null <

Install Bind9

BIND9 Installation BASH 
sudo apt install bind9 bind9utils bind9-doc -y
sudo systemctl enable --now bind9
sudo systemctl status bind9

named.conf.options (forwarders)

/etc/bind/named.conf.options INI 
options {
  directory "/var/cache/bind";
  recursion yes;
  allow-query { any; };

  forwarders {
    8.8.8.8;
    8.8.4.4;
  };

  dnssec-validation auto;
  auth-nxdomain no;
  listen-on { 127.0.0.1; 10.10.5.1; };
};

Create zone files

Zone File Creation BASH 
cd /etc/bind
sudo cp db.local db.kelompok5
sudo cp db.127 db.10

Register zones in named.conf.local

/etc/bind/named.conf.local INI 
zone "kelompok5.sch.id" {
  type master;
  file "/etc/bind/db.kelompok5";
};

zone "10.10.5.in-addr.arpa" {
  type master;
  file "/etc/bind/db.10";
};

db.kelompok5 example

/etc/bind/db.kelompok5 DNS 
$TTL 604800
@   IN  SOA ns1.kelompok5.sch.id. admin.kelompok5.sch.id. (
        2024080101 ; serial
        604800
        86400
        2419200
        604800 )
@       IN  NS  ns1.kelompok5.sch.id.
ns1     IN  A   10.10.5.1
www     IN  A   10.10.5.11
@       IN  A   10.10.5.11

db.10 example (reverse)

/etc/bind/db.10 DNS 
$TTL 604800
@   IN  SOA ns1.kelompok5.sch.id. admin.kelompok5.sch.id. (
        2024080101 ; serial
        604800
        86400
        2419200
        604800 )
@       IN  NS      ns1.kelompok5.sch.id.
1       IN  PTR     ns1.kelompok5.sch.id.
11      IN  PTR     www.kelompok5.sch.id.

Validation & restart

BIND9 Validation BASH 
sudo named-checkconf
sudo named-checkzone kelompok5.sch.id /etc/bind/db.kelompok5
sudo named-checkzone 10.10.5.in-addr.arpa /etc/bind/db.10
sudo systemctl restart bind9
sudo systemctl status bind9

Testing

DNS Testing Commands BASH 
nslookup www.kelompok5.sch.id 10.10.5.1
dig @10.10.5.1 www.kelompok5.sch.id
nslookup 10.10.5.11 10.10.5.1
dig -x 10.10.5.11 @10.10.5.1

Troubleshooting tips

  • Periksa named-checkzone output untuk error pada SOA/serial.
  • Pastikan permissions file zona dan ownership benar (root:bind).
  • Jika menggunakan firewall, buka port 53 tcp/udp.

Instalasi & Konfigurasi Web Server (Apache2)

Panduan membuat web server sederhana menggunakan Apache2, virtual host, rewrite, SSL (Let's Encrypt), dan langkah testing.

Prerequisites

  • OS: Ubuntu/Debian (tested on Ubuntu 22.04)
  • Root/sudo access
  • DNS server already configured (from DNS section)
  • Domain pointing to your server IP

Install Apache

Apache2 Installation BASH 
sudo apt update
sudo apt install apache2 -y
sudo systemctl enable --now apache2
sudo systemctl status apache2

Firewall Configuration

UFW Firewall Setup BASH 
sudo ufw app list
sudo ufw allow 'Apache Full'
sudo ufw allow 'OpenSSH'
sudo ufw enable
sudo ufw status

Create site directory & sample page

Site Directory Setup BASH 
sudo mkdir -p /var/www/kelompok5.sch.id/public
sudo chown -R $USER:$USER /var/www/kelompok5.sch.id
sudo chmod -R 755 /var/www/kelompok5.sch.id

# Create sample HTML page
sudo tee /var/www/kelompok5.sch.id/public/index.html > /dev/null <


    
    
    Kelompok 5 - DNS & Web Server


    
Selamat Datang di Kelompok5.sch.id

    
Web server berhasil dikonfigurasi!



EOF

VirtualHost Configuration

/etc/apache2/sites-available/kelompok5.sch.id.conf APACHE 
sudo tee /etc/apache2/sites-available/kelompok5.sch.id.conf > /dev/null <
    ServerName kelompok5.sch.id
    ServerAlias www.kelompok5.sch.id
    DocumentRoot /var/www/kelompok5.sch.id/public
    
    # Directory configuration
    
        Options Indexes FollowSymLinks MultiViews
        AllowOverride All
        Require all granted
    
    
    # Security headers
    Header always set X-Content-Type-Options nosniff
    Header always set X-Frame-Options DENY
    Header always set X-XSS-Protection "1; mode=block"
    
    # Logging
    ErrorLog \${APACHE_LOG_DIR}/kelompok5_error.log
    CustomLog \${APACHE_LOG_DIR}/kelompok5_access.log combined
    LogLevel warn

EOF

Enable Site & Required Modules

Enable Apache Site & Modules BASH 
# Enable required modules
sudo a2enmod rewrite
sudo a2enmod headers
sudo a2enmod ssl

# Enable site and disable default
sudo a2ensite kelompok5.sch.id.conf
sudo a2dissite 000-default.conf

# Test configuration and reload
sudo apache2ctl configtest
sudo systemctl reload apache2

Permissions & Security

File Permissions

  • Set owner: sudo chown -R www-data:www-data /var/www/kelompok5.sch.id
  • Set dir perms: find /var/www/kelompok5.sch.id -type d -exec chmod 755 {} \;
  • Set file perms: find /var/www/kelompok5.sch.id -type f -exec chmod 644 {} \;
  • Secure .htaccess: chmod 644 /var/www/kelompok5.sch.id/public/.htaccess

SSL/HTTPS Setup with Let's Encrypt

Certbot SSL Installation BASH 
# Install Certbot
sudo apt install certbot python3-certbot-apache -y

# Obtain SSL certificate (requires valid domain)
sudo certbot --apache -d kelompok5.sch.id -d www.kelompok5.sch.id

# Test auto-renewal
sudo certbot renew --dry-run

# Check certificate status
sudo certbot certificates

HTTPS VirtualHost (Auto-generated by Certbot)

SSL VirtualHost Configuration APACHE 

    ServerName kelompok5.sch.id
    ServerAlias www.kelompok5.sch.id
    DocumentRoot /var/www/kelompok5.sch.id/public
    
    # SSL Configuration
    SSLEngine on
    SSLCertificateFile /etc/letsencrypt/live/kelompok5.sch.id/fullchain.pem
    SSLCertificateKeyFile /etc/letsencrypt/live/kelompok5.sch.id/privkey.pem
    Include /etc/letsencrypt/options-ssl-apache.conf
    
    # Security Headers
   
    Header always set X-Content-Type-Options nosniff
    Header always set X-Frame-Options DENY

Performance Optimization

Optimization Tips

  • Enable compression: sudo a2enmod deflate
  • Enable caching: sudo a2enmod expires
  • Configure keepalive in /etc/apache2/apache2.conf
  • Monitor with: apache2ctl status

Troubleshooting Common Issues

  • 403 Forbidden: Check directory permissions and ownership
  • 500 Internal Error: Check /var/log/apache2/error.log
  • Site not loading: Verify DNS resolution and VirtualHost config
  • SSL issues: Check certificate validity with sudo certbot certificates
  • Port conflicts: Check with sudo netstat -tlnp | grep :80

Instalasi & Konfigurasi WordPress (Ubuntu 22 + Apache2 + MySQL)

Panduan lengkap setup WordPress di web server Ubuntu 22.04 menggunakan Apache2, MySQL/MariaDB, PHP, konfigurasi VirtualHost, dan pengaturan template.

Prerequisites

  • OS: Ubuntu 22.04 (server/VM)
  • Apache2 sudah terinstal & jalan (lihat materi Web Server)
  • MySQL/MariaDB server
  • PHP 8.x + ekstensi
  • Domain sudah diarahkan ke server

Install LAMP Stack (Apache, MySQL, PHP)

LAMP Installation BASH 
sudo apt update && sudo apt upgrade -y
sudo apt install apache2 mysql-server -y
sudo apt install php libapache2-mod-php php-mysql php-cli php-curl php-xml php-mbstring unzip -y

# Check PHP version
php -v

Setup MySQL Database

MySQL Database for WordPress SQL 
sudo mysql -u root

-- Buat database dan user WordPress
CREATE DATABASE wordpress DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;
CREATE USER 'wpuser'@'localhost' IDENTIFIED BY 'passwordku';
GRANT ALL PRIVILEGES ON wordpress.* TO 'wpuser'@'localhost';
FLUSH PRIVILEGES;
EXIT;

Download & Extract WordPress

WordPress Download BASH 
cd /tmp
wget https://wordpress.org/latest.zip
unzip latest.zip

# Pindah ke folder web
sudo mkdir -p /var/www/kelompok5.sch.id/public
sudo cp -R wordpress/* /var/www/kelompok5.sch.id/public

# Set ownership & permission
sudo chown -R www-data:www-data /var/www/kelompok5.sch.id
sudo find /var/www/kelompok5.sch.id -type d -exec chmod 755 {} \;
sudo find /var/www/kelompok5.sch.id -type f -exec chmod 644 {} \;

Konfigurasi WordPress

wp-config.php setup PHP 
cd /var/www/kelompok5.sch.id/public
cp wp-config-sample.php wp-config.php

# Edit file
sudo nano wp-config.php

# Ubah bagian DB sesuai database tadi:
define( 'DB_NAME', 'wordpress' );
define( 'DB_USER', 'wpuser' );
define( 'DB_PASSWORD', 'passwordku' );
define( 'DB_HOST', 'localhost' );

VirtualHost Apache untuk WordPress

/etc/apache2/sites-available/kelompok5.sch.id.conf APACHE 
<VirtualHost *:80>
    ServerName kelompok5.sch.id
    ServerAlias www.kelompok5.sch.id
    DocumentRoot /var/www/kelompok5.sch.id/public

    <Directory /var/www/kelompok5.sch.id/public>
        AllowOverride All
        Require all granted
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/wp_error.log
    CustomLog ${APACHE_LOG_DIR}/wp_access.log combined
</VirtualHost>

Aktifkan Site & Restart Apache

Enable & Restart Apache BASH 
sudo a2enmod rewrite
sudo a2ensite kelompok5.sch.id.conf
sudo a2dissite 000-default.conf
sudo apache2ctl configtest
sudo systemctl reload apache2
